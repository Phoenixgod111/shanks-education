# Спецификация: онбординг (вариант A) и раздел «Предметы» (вариант 1 + хедер)

**Дата:** 2026-03-29  
**Статус:** согласовано с продуктом  
**Контекст:** Shanks (`index.html`, `js/app.js`, `js/data.js`), избранное `favoritesByGrade` в `localStorage`.

## Цели

1. При первом входе провести короткий онбординг и на его основе выставить класс и стартовый набор избранных предметов.
2. На вкладке «Предметы» убрать переключатель классов; показывать класс в шапке и две зоны контента — «мои предметы» и возможность добавить остальные через явное действие.

## 1. Онбординг (вариант A — короткий мастер)

### Шаги

1. **Класс** — выбор одного значения из `D.grades` (5–11).
2. **Предметы** — мультивыбор из предметов каталога выбранного класса (`catalogByGrade[grade]`). Минимум один предмет для завершения (или мягкий минимум с подсветкой кнопки «Далее» — зафиксировать при реализации; рекомендация: минимум 1).
3. **Опционально** — один экран «цель» (короткий CTA), можно пропустить или объединить с финальным «Готово».

### Поведение после завершения

- Записать в `prefs`: `onboardingCompleted: true`, `grade` (или только в `state` + отдельный ключ `userGrade` в prefs — предпочтение: **единый источник**: сохранять выбранный класс в prefs, например `prefs.grade`, и синхронизировать с `state.grade` при старте).
- Инициализировать `prefs.favoritesByGrade[class]` выбранными id предметов (остальные классы в `favoritesByGrade` могут оставаться пустыми до смены класса в настройках).
- Скрыть мастер; показать основное приложение.

### Повторный запуск / миграция

- Пользователи с уже заполненным `prefs.initialized` и без флага онбординга: трактовать как завершивших онбординг **или** один раз показать «обновление» — отдельное решение; минимум — если нет `onboardingCompleted`, показать мастер или мягко предложить в настройках.

## 2. Раздел «Предметы» (вариант 1 + хедер)

### Шапка страницы

Порядок сверху вниз:

1. Строка класса в человекочитаемом виде: **«{N} класс»** (например, `7 класс`). Можно оставить визуальный акцент как у текущего kicker, но текст — именно такой, без лишних символов в title если не нужны по дизайну.
2. Заголовок секции / страницы: **«Мои предметы»** (вместо общего «Предметы» на этом экране, если продуктово ок заменить; иначе подзаголовок под классом).

Удалить из этой страницы блок **pills** выбора класса (`index.html` строки с `data-grade`).

### Зоны контента

1. **Мои предметы** — карточки только из текущего `favoritesByGrade[state.grade]` (как сейчас `#fav-list`), порядок как в избранном или как в каталоге — оставить текущую логику сортировки.
2. **Кнопка «Добавить предмет»** — первичное действие (например, под списком или в шапке контента). По нажатию: модальное окно или полноэкранный список **всех** предметов класса с чекбоксами: уже в избранном — отмечены; сохранение обновляет `favoritesByGrade` и перерисовывает список.
3. Не показывать на этой вкладке полный каталог «все предметы с сердечками» по умолчанию — только поток «мои» + добавление (соответствует варианту 1: два явных режима — список избранного и расширение через действие).

Пустое состояние «мои предметы»: текст + кнопка «Добавить предмет» (ведёт в тот же picker).

## 3. Смена класса

- Переключатель класса **не** на вкладке «Предметы».
- Смена класса — в **настройках** (отдельный экран или секция): при смене обновить `state.grade` / `prefs.grade` и показывать избранное для нового класса (списки по-прежнему per-grade в `favoritesByGrade`).

## 4. Технические точки интеграции

- `renderSubjects()` в `app.js`: убрать обновление pills; обновлять хедер (`sub-grade-kicker` → текст `{grade} класс`, добавить/обновить элемент для «Мои предметы»); не рендерить полный `#sub-rows` на главном виде — только fav + CTA; логика picker — новый UI слой.
- Онбординг: новый view/overlay в `index.html` или один экран с шагами в JS; по завершении выставить grade + favorites + флаг.
- Ключи `localStorage`: расширить существующий `PREFS_KEY` объект (например `onboardingCompleted`, `grade`).

## 5. Критерии готовности

- [ ] Новый пользователь проходит мастер и видит на «Предметы» класс в шапке и выбранные предметы без pills.
- [ ] «Добавить предмет» добавляет/снимает предметы с избранного и синхронизирует главную.
- [ ] В настройках можно сменить класс; вкладка «Предметы» отражает новый класс.
